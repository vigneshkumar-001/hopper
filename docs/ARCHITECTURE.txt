================================================================================
                           SYSTEM COMPONENTS
================================================================================

1. LOGGER SERVICE (Core)
   ├── File: lib/Core/Services/logger_service.dart
   ├── Responsibility: Main logging engine
   ├── Functions:
   │   ├── log() - Basic logging
   │   ├── logError() - Error logging with stack trace
   │   ├── logApiRequest() - API request logging
   │   ├── logApiResponse() - API response logging
   │   ├── logApiError() - API error logging
   │   ├── logSocketEvent() - Socket event logging
   │   ├── logNavigation() - Navigation logging
   │   ├── logDeviceInfo() - Device info logging
   │   ├── logAppCrash() - App crash logging
   │   ├── exportLogs() - Export to file
   │   ├── getLogsContent() - Get logs as string
   │   ├── clearLogs() - Delete all logs
   │   └── getLogSize() - Get file size
   └── Status: ✅ Complete

2. API LOGGER INTERCEPTOR
   ├── File: lib/api/interceptors/api_logger_interceptor.dart
   ├── Responsibility: Auto-log all API traffic
   ├── Extends: Dio Interceptor
   ├── Methods:
   │   ├── onRequest() - Log requests
   │   ├── onResponse() - Log responses
   │   └── onError() - Log API errors
   ├── Integration: Added to all Dio instances in request.dart
   └── Status: ✅ Complete

3. SOCKET LOGGER UTIL
   ├── File: lib/Core/Services/socket_logger_util.dart
   ├── Responsibility: Utilities for socket logging
   ├── Methods:
   │   ├── setupSocketLogging() - Setup auto-logging
   │   └── emitWithLogging() - Emit with logging
   ├── Integration: Manual setup in socket code
   └── Status: ✅ Complete

4. DEVELOPER LOGS SCREEN
   ├── File: lib/Presentation/Drawer/screens/dev_logs_screen.dart
   ├── Responsibility: UI for viewing/managing logs
   ├── Features:
   │   ├── Terminal-style log viewer
   │   ├── Export button (📥)
   │   ├── Copy button (📋)
   │   ├── Clear button (🗑️)
   │   ├── Log file size display
   │   └── Real-time updates
   ├── Integration: Accessed via 🐛 button
   └── Status: ✅ Complete

5. RIDE ACTIVITY INTEGRATION
   ├── File: lib/Presentation/Drawer/screens/ride_activity.dart
   ├── Responsibility: Provide access to logs
   ├── Integration: 🐛 button in top-right
   ├── Action: Opens DevLogsScreen on tap
   └── Status: ✅ Complete

  │

  │     ├─ FlutterError.onError
  │     └─ runZonedGuarded
  │         └─ Both log crashes to file
  │
  ├─ 7. Firebase.initializeApp()
  │
  ├─ 8. Register FCM Handler
  │
  ├─ 9. Start App
  │     └─ runApp(MyApp())
  │         ├─ Logging system is now active
  │         └─ All API calls auto-logged
  │
  ├─ 10. Set UI Overlay Style
  │
  └─ 11. Init FCM Notifications

RUNNING
  │
  ├─ User makes API call
  │  └─ Logged by ApiLoggerInterceptor
  │
  ├─ Socket event occurs
  │  └─ Logged by SocketLoggerUtil (if setup)
  │
  ├─ User navigates
  │  └─ Logged by RouteLogger (if setup)
  │
  └─ App crashes
     └─ Logged by Error Handler
